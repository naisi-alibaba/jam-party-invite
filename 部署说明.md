# 服务器部署速记（git pull 版）

前端页面你已部署。现在同一个仓库里多了 `server.js`（后端 + 静态托管）。

## 在服务器上
```bash
cd 你的仓库目录        # git pull 后，里面有 index.html、img/、music/、server.js
git pull

# 装 pm2（首次）
npm i -g pm2

# 用 80 端口启动（需 root/sudo）；ADMIN_KEY 自己设一个
PORT=80 ADMIN_KEY=你的口令 pm2 start server.js --name jamparty
pm2 save && pm2 startup      # 开机自启，按提示再执行它打印的命令
```

打开 `http://你的IP/`，走一遍填名挑瓶入场，滑到底看果酱墙是否出现瓶子。

## 常用
- 看日志：`pm2 logs jamparty`
- 改了 server.js 后：`pm2 restart jamparty`
- 只改了 index.html：`git pull` 即可，静态实时读，无需重启

## 看名单 / 导出
- 全量：`http://你的IP/api/admin?key=你的口令`
- CSV：`http://你的IP/api/admin?key=你的口令&format=csv`

## 安全 & 数据
- `server.js`、`guests.json`、`部署说明.md` 已被后端**屏蔽**，外部访问返回 403，名单不会被下载。
- 报名数据存 `guests.json`（已加入 .gitignore，不会误传回 GitHub）。删掉它=清空。

## 端口
- 用 80 需 root；若不想用 root，可 `PORT=3000 pm2 start ...` 起在 3000，再用 Nginx 反代 80→3000（可选）。
- 腾讯云轻量控制台记得放行对应端口。
